## Architecture

### Routing

Uses **TanStack Router** with file-based routing. The Vite plugin auto-generates `src/routeTree.gen.ts` from `src/routes/` — never edit that file manually. Route layout:

- `__root.tsx` — wraps everything; adds `<Toaster />`
- `_authenticated.tsx` — layout route with JWT guard (`beforeLoad` redirects to `/sign-in` if no token) and the `AppSidebar` layout
- `_authenticated/` — all protected feature routes (dashboards, queries, components, data-sources, settings)
- `sign-in.tsx` / `sign-up.tsx` — public routes

### Data fetching

**TanStack Query** (`@tanstack/react-query`) with `staleTime: 30_000` and 1 retry. The `QueryClient` is passed as router context so loaders can prefetch. All API calls go through `src/services/api.ts` — an Axios instance that attaches the Bearer token from `localStorage` and redirects to `/sign-in` on 401.

Base URL: `VITE_API_URL` env var (defaults to `http://localhost:3000`).

### UI

**shadcn/ui** with the **nova** preset. Components are in `src/components/ui/` (generated by shadcn CLI — edit freely but don't run `shadcn add` without checking for conflicts). Path alias `@/` maps to `src/`.

### Key custom components

- `src/components/query-builder/QueryBuilder.tsx` — visual MongoDB aggregation builder; emits `QueryConfiguration` objects matching the backend's `QueryBuilderService` input schema
- `src/components/visualizations/ChartRenderer.tsx` — recharts renderer for TABLE / BAR_CHART / LINE_CHART / PIE_CHART / KPI component types
- `src/components/visualizations/ComponentEditor.tsx` — wraps `QueryBuilder` + `ChartRenderer` for the create/edit component flow
- `src/components/dashboard/DashboardGrid.tsx` — react-grid-layout v2 grid; uses new v2 API (`gridConfig`, `dragConfig`, `resizeConfig` props)

### Types

All shared TypeScript interfaces are in `src/types/index.ts`. These mirror the backend's Prisma models and API response shapes.
